Part A · Trial setup
What this pilot is designed to prove
The goal is not just to improve aggregate visibility. The goal is to show whether Edge optimisation changes measurable outcomes for a controlled set of pages, with prompt coverage aligned tightly enough that page-level lift can be attributed with confidence.

Why this setup is cleaner

  • It avoids letting broad prompts over-credit homepage and attraction hubs.
  • It creates explicit measurement cohorts for visa, transport, Muslim travel, and The Peak.
  • It keeps the prompt pool small enough to compare consistently before and after rollout.
  • It preserves a realistic mix of planning, discovery, logistics, and destination-specific intent.

Current risk in the uploaded prompt file

  • The existing 200 prompts are strong for broad visitor discovery.
  • They are weak for page-specific attribution.
  • Two selected pages have zero explicit coverage today: Muslim travel and The Peak.
  • Visa and getting around are materially under-represented versus their trial importance.

Part B · Trial content
The balanced 50-prompt pack
Each selected page gets exactly five prompts. The prompts are written to reduce overlap and improve page-level comparability between the pre-Edge and post-Edge runs.
B1 Customer-shareable 50 prompt set

Use this exact set before and after Edge optimisation. Do not swap prompts mid-test.

1. Homepage

  1. What is Discover Hong Kong and what should first-time visitors use it for?

B2 How to measure the trial

Keep the run conditions as fixed as possible. The goal is to measure Edge impact, not prompt drift.

Primary comparison metrics

  • Times Cited by selected page
  • Prompts Cited In by selected page
  • Owned citation share across the 50 prompts
  • LLM referral traffic to selected pages
  • Bounce rate / engagement quality on referred visits
  • Content visibility / gain ratio where rerun exports are available

Best way to read the results

  • Read results first at the page cohort level, then at the total trial level.
  • Look for movement in both absolute counts and share of prompts won.
  • Separate hub-page lift from leaf-page lift.
  • Call out pages that already had traffic versus pages that had little or no owned citation presence before Edge.

Part C · Operating rules
Keep the test controlled
A clean trial depends as much on discipline as on page selection. The rules below keep the before/after readout fair.
C1 Recommended operating rules

These rules are what make this pack customer-safe and analytically useful.

01

Use the same 50 prompts before and after

No prompt additions, deletions, or rewrites during the trial window.

02

Keep model and market fixed

Use the same model, same market setting, and the same test configuration for both runs.

03

Report at page level first

Start with each selected URL's 5-prompt cohort, then roll up to the total.

04

Separate visibility from engagement

Citation lift and referral lift are not the same as on-site engagement lift; report both.

05

Flag pages with prior referrals

Food & drink, visa information, and Muslim travel already had LLM visits, so compare them differently from zero-baseline pages.

06

Keep customer readout simple

Show which pages improved, by how much, and whether those gains turned into visits or better engagement.

Recommended headline for the customer: "We intentionally rebalanced the prompt set so each selected page has its own measurable cohort. That makes the before/after result more attributable to Edge optimisation and less dependent on prompt mix."
