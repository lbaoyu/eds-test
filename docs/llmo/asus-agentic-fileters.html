Agentic Traffic 指南

了解 Agent Type 與 User Agent 篩選器

本指南說明分析 AI 流量時最常使用的兩個篩選器：Agent Type 與 User Agent。 內容以客戶為對象，使用容易理解的方式解釋每個選項的含義。

Agent Type = 大類別 User Agent = 特定機器人或代理名稱 客戶友善說明

總覽

這兩個篩選器回答的是不同的問題：

Agent Type

幫助您了解目前看到的是哪一種類型的 AI 行為， 例如模型訓練、搜尋爬取、即時問答擷取、研究分析或任務執行。

User Agent

幫助您辨識提出請求的特定 AI 機器人或代理， 例如 GPTBot、ChatGPT-User、ClaudeBot 或 PerplexityBot。

1) Agent Type — 各選項代表什麼

Agent Type 會依照代理執行的角色，把 AI 流量分成較大的類別。

Agent Type 選項 含義 常見範例
All Agents 顯示所有已辨識的 AI 相關代理，不限制特定類別。 所有已知的 AI 機器人與代理
Training Bots 蒐集公開網頁內容，以協助改進未來 AI 模型的代理。 GPTBot、ClaudeBot
Web Search Crawlers 發現並索引內容，讓內容可出現在 AI 輔助搜尋體驗中的代理。 OAI-SearchBot、PerplexityBot、Claude-SearchBot
Chatbots 在使用者即時提問時擷取內容，讓 AI 可以回答問題或引用來源的代理。 ChatGPT-User、Perplexity-User、Claude-User、MistralAI-User
Research Agents 設計用來執行更深入、多步驟研究與資訊整合的代理。 Gemini-Deep-Research、Google-NotebookLM
Media Fetchers 請求圖片、圖示或其他媒體資產，而非一般頁面內容的代理。 AI 平台使用的用戶端或媒體擷取變體
Action Agents 可以代表使用者在工具或瀏覽器中執行目標導向操作的代理。 GoogleAgent-Chrome、GoogleAgent-Mariner
Shopping Agents 支援購物相關任務、比較或商品導向操作的代理。 GoogleAgent-Shopping

最簡單的理解方式： Agent Type 告訴您的是 這個 AI 系統正在做什麼工作，而不只是它的名稱。

2) User Agent — 各選項代表什麼

User Agent 顯示的是提出請求的特定機器人或代理身分。

User Agent 選項 提供者 常見角色 白話說明
GPTBot OpenAI Training Bot 用來蒐集公開網頁內容，可能用於改進未來 AI 模型。
OAI-SearchBot OpenAI Web Search Crawler 用於搜尋相關的內容發現與擷取。
ChatGPT-User OpenAI Chatbot 當 ChatGPT 代表使用者問題去擷取內容時所使用。
PerplexityBot Perplexity Web Search Crawler 用來為 Perplexity 的搜尋與回答體驗索引及發現內容。
Perplexity-User Perplexity Chatbot 當 Perplexity 為了回答使用者問題而即時抓取內容時所使用。
ClaudeBot Anthropic Training Bot 用來蒐集公開內容，以改進模型。
Claude-SearchBot Anthropic Web Search Crawler 用來為搜尋型回答索引公開內容。
Claude-User Anthropic Chatbot 當 Claude 因為使用者要求而造訪某個頁面擷取內容時所使用。
Gemini-Deep-Research Google Research Agent 用於更深入、多步驟的研究任務，以蒐集並整合資訊。
MistralAI-User Mistral AI Chatbot 當 Mistral 在對話中即時擷取網頁內容時所使用。
GoogleAgent-Chrome Google Action Agent 當 Google 的 AI 代理執行瀏覽器或工具型操作時所使用。
GoogleAgent-Mariner Google Action Agent 用於任務導向的瀏覽與執行流程。

最簡單的理解方式： User Agent 告訴您 到底是哪一個 AI 機器人提出了這次請求。

3) 這兩種篩選器有什麼不同

篩選器 最適合用來回答的問題
Agent Type 了解目前發生的是哪一種類型的活動，例如模型訓練、搜尋發現、即時回答產生或任務執行。
User Agent 了解是哪一個特定機器人負責該活動。

簡單總結： Agent Type 說明的是行為類別， 而 User Agent 指出的是執行該行為的具體機器人或代理。

4) 實用補充說明

  • 不是每個供應商都只使用一種機器人。同一個供應商可能會針對訓練、搜尋、即時擷取、媒體存取或任務執行使用不同代理。
  • 新的 AI 代理會隨時間持續出現，因此可用選項也可能隨著平台推出新能力而擴充。
  • 如果您在檢視流量趨勢，通常先用 Agent Type 會更容易掌握整體策略方向；如果要查看特定供應商或機器人的細節，再用 User Agent 會更合適。

5) 快速摘要

當您想回答以下問題時，請使用 Agent Type：

  • 這些請求主要來自訓練型機器人、聊天型機器人，還是搜尋爬蟲？
  • 我的 AI 流量中，有多少屬於研究導向，又有多少屬於使用者即時擷取？

當您想回答以下問題時，請使用 User Agent：

  • 這些流量是來自 ChatGPT、Claude、Perplexity、Gemini，還是其他供應商？
  • 實際活躍的是哪一個機器人？
