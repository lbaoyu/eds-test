Edge Delivery Services (EDS): Block Collection vs Block Party

This document explains the difference between Block Collection and Block Party in Adobe Experience Manager Edge Delivery Services and is formatted for easy export to PDF.

1.Overview

In Edge Delivery Services (EDS), Block Collection and Block Party are related but distinct concepts:

  • Block Collection is the curated, reusable library of blocks (the EDS equivalent of AEM Core Components).
  • Block Party is the public showcase and explorer for that collection (documentation, examples, and integrations).

Use Block Collection when you want code to reuse.
Use Block Party when you want to discover, evaluate, and understand what’s available.

2.What is the Block Collection?

The Block Collection is the official, community‑driven set of reusable blocks for Edge Delivery Services projects.

2.1Purpose

  • Acts as the equivalent of Core Components in traditional AEM.
  • Provides a set of pre‑built, extensible blocks (hero, cards, table, accordion, etc.) so teams don’t have to rebuild common patterns from scratch.
  • Designed as a modern, customizable foundation for EDS sites.

“The Block Collection serves as the equivalent of Core Components in Edge Delivery Services. The collection provides a set of pre-built, extensible blocks designed for modern websites, providing an easily customizable foundation for any project.”

2.2How it is used

Typical usage pattern for the Block Collection:

  1. Start from the Block Collection boilerplate and/or the adobe/aem-block-collection GitHub repository.
  2. Copy the required block folders (for example accordion, cards, table) into your EDS project under blocks/.
  3. Adjust the CSS and JS to match your project’s design and behavior.
  4. Expose those blocks to authors (via document‑based authoring, Universal Editor, or custom block libraries).

3.What is Block Party?

Block Party is the experience built around the collection: a showcase, catalog, and exploration site for EDS blocks and integrations.

3.1Purpose

  • Provides a visual catalog of blocks and patterns available for Edge Delivery Services.
  • Used to showcase:
    • Individual blocks (layout, content structure, and authoring).
    • Framework integrations (for example Adobe Target, Commerce drop‑ins).
    • Example implementations contributed by Adobe and the wider developer community.

“The Block Collection is a place for the AEM developer community to showcase what they have built on AEM sites. It also allows others to avoid reinventing the wheel and reuse these blocks / code snippets / integrations built by the community and tweak the code as necessary to fit their own projects. See Block Party for everything it has to offer. Note: While we love and support our AEM developer community, Adobe is not responsible for maintaining or updating the code that is showcased in Block Party. Please use the code at your own discretion.”

3.2How it is used

You use Block Party to:

  • Discover what blocks exist and how they behave.
  • Inspect examples of markup and behavior before reusing or adapting them.
  • Find integrations (for example “Framework integrations” such as Target exports and Commerce) that have been contributed and documented.

Internal Slack discussions refer to Block Party in this way, for example when pointing to a newly contributed Target integration block under “Framework integration”.

4.Key Differences at a Glance

4.1Conceptual difference

Block Collection

  • Library of blocks.
  • Source of reusable code and patterns.
  • Consider it the “Core Components for EDS”.

Block Party

  • Showcase & explorer for that library.
  • Source of examples, documentation, and integration demos.
  • Not a guarantee of long‑term maintenance for every sample.

4.2Comparison table

Aspect Block Collection Block Party
Primary role Canonical library of reusable EDS blocks. Showcase / gallery site for blocks and integrations.
Analogy Like Core Components in classic AEM. Like a component catalog / pattern library site.
What you get Boilerplate, code repos, ready‑to‑copy block implementations. Demos, documentation, block examples, and integration examples.
Typical usage Copy blocks into your project and customize CSS/JS. Browse patterns, learn usage, then copy/adapt into your own codebase.
Maintenance Treated as a base for projects; updated via repo/boilerplate. “Use at your own discretion”; Adobe is not responsible for keeping every showcased sample updated.
Documentation link Block Collection Linked from Block Collection docs as “See Block Party for everything it has to offer”.

5.How to Decide Which to Reference

When talking about reusable EDS building blocks for implementations, refer to “Block Collection”.

Example:

“We’ll base our hero and table blocks on the EDS Block Collection to avoid custom one‑off implementations.”

When pointing authors or developers to explore available patterns, refer to “Block Party”.

Example:

“You can see how that integration works in Block Party, then copy the block into your project from the collection.”
