Prompt quality reference · Adobe LLM Optimizer / Adobe Brand Visibility

Quality attributes for a prompt worth tracking.

In Adobe Brand Visibility a prompt is a test case, not a keyword. One rule gates everything, then quality sits at two levels: the shape of the whole set, and the build of each prompt in it.

Gate: real, trusted sources Set level: structure & coverage Prompt level: how each one is built
The idea to hold onto

Every number you report inherits the quality of its prompt

Brand mentions, competitor presence, citations and sentiment are all read off what the models say back. If a prompt leaves the model room to guess, the answer changes from one run to the next — and that change looks like a ranking shift when it is really just noise. Most teams start by treating prompts like search keywords; moving away from that is the single biggest step in getting this right.

A prompt is closer to a software test case than a search term

A good test case is realistic, specific, and repeatable. Run the same loosely-written prompt a few times and the list of brands it returns keeps changing — often with almost no chance of getting the same set twice. That steadiness has to come from the prompt, not the model. That is the whole reason to be deliberate about quality.

Gate 0 · Prerequisite · applies before either lens

Every tracked prompt starts from real, trusted demand

Before you ask whether the set is well-structured or a prompt is well-built, ask a simpler question: where did this prompt come from? Prompts invented on a whiteboard measure what your team imagines people ask. Prompts drawn from real query data measure what people actually ask — and only the second kind produces a signal worth acting on.

Ground every prompt in an evidence source: the Semrush real-query panel (hundreds of millions of real AI prompts), Google Search Console, your own on-site search, competitor-gap research, and the industry prompt library as a starting point. This gate sits above both lenses — a beautifully structured set of made-up prompts is still made up.

Two lenses

Judge the set and the prompt separately

Once the sourcing gate is met, the attributes split cleanly in two. One lens asks whether the set, as a whole, tests the right things. The other asks whether a single prompt is well built. A prompt can be perfectly built and still be useless if the set around it has no coverage — and a well-covered set can still be noisy if the individual prompts are sloppy.

Lens A — The set

Structure, grouping & coverage

"Does the set, taken as a whole, test the right things — and can I read a steady signal off it?"

Judged across Categories, Topics and Prompts. About hierarchy, grouping by intent, deliberate coverage across journeys, personas and strategic tiers, enough prompts per topic, clear ownership, and keeping the set fresh.

Lens B — The prompt

How one prompt is built

"Is this one prompt a clean, realistic question that runs the same way every time?"

Judged one prompt at a time. About being a single, realistic question in your customers' own words, aimed at a real decision, specific and self-contained enough to hold steady across runs — then filed under the right topic and wired so a poor result leads somewhere.

Scope & honesty

A tracked prompt is one turn, not a conversation

Real AI use is multi-round — people ask, refine, and follow up. Adobe Brand Visibility, like other tools in this category, runs every prompt single-shot: one standalone query, sent fresh and repeated over time. That's a deliberate design choice — it keeps each measurement repeatable and easy to trace — not a shortcoming to hide. It does mean it's worth being clear about where the signal ends.

What a single prompt does capture
  • The decisive moment, with intent fully specified"given someone who wants exactly this, does the brand surface?"
  • A clean, repeatable reading you can traceno earlier turns to muddy which input caused the mention
  • Where you win or lose across the journeywhen the funnel is modelled as separate prompts
What it can't model today
  • Path dependencewhether a follow-up question flips the recommendation
  • Memory & personalisationcontext a logged-in user has built up over time
  • The model's own clarifying questionsthe back-and-forth before it commits to an answer
Three ways to approximate multi-turn without leaving the tool

Collapse the conversation into one prompt. Bake the context a user would build over several turns into a single self-contained question — which is exactly what Specific and Self-contained are for. Prompt 5, 6

Sequence the funnel as separate prompts. Track awareness → consideration → decision as distinct prompts; you lose the causal chain but see where you drop out. Set 3

Chain it offline, track the endpoint. Run a 2–3 turn conversation in a chat interface or against the model's API, then add the representative final-turn question back as a tracked prompt.

Lens A

Set-level attributes — the whole library

These describe the collection, not any single prompt. They decide whether the results roll up into something you can read and act on. Start with what the set is for.

Compose across strategic tiers

A good set is a deliberate blend, not one note

Every prompt is doing a strategic job. A healthy set spans more than one — a common split is three tiers. The weighting between them is a choice you make to fit a business goal, the same way the branded / unbranded mix is (see below). There's no default ratio worth copying.

Recover
Win back lost ground

Questions where competitors are cited and you're absent, or where you used to show up and slipped. The point is to close visible gaps.

Expand
Reach new demand

Questions in adjacent topics or intents you don't target yet, where there's real demand. The point is to grow into new territory.

Defend
Hold what you've won

Questions where you already appear and get cited. The point is to protect and strengthen that presence so you don't lose it.

Set · 1

Sound structure

Three levels: Categories (broad business areas) hold Topics (groups of related questions) hold Prompts. Pick one way to define categories and stick with it — by business unit, by URL section, or by product line.

Guide: 5–10 categories. Changing how categories are defined wipes your history, so decide before you scale up.
Set · 2

Grouped by intent

Group prompts by what the person is trying to do, not by how your site or team is organised. Name topics by the kind of question so they work across every category.

ComparisonHow-toPricingTroubleshootingReviews

Keep the topic list short — roughly 3–10 per category.
Set · 3

Intentional coverage

The set should deliberately span the buying journey, the personas and markets you care about, the strategic tiers above, and a chosen mix of branded and unbranded questions.

Treat each mix as a lever, not a default. Set the branded / unbranded split to serve a goal — defending brand perception pulls it one way, chasing demand where you're invisible pulls it the other. What matters is that you can justify the mix, not that it hits a template number.

The right mix is the one that maps to a business goal you can name.
Set · 4

Enough prompts per topic

A topic needs enough prompts that one odd answer can't swing the whole trend. With enough of them, many runs average out into a reading you can trust. Thin topics give you noise dressed up as movement.

On topics that matter: 15 at a minimum, 20–30 ideally. When pulling prompts in bulk for breadth, ~3 per topic to spread reach.
Set · 5

No duplication

Models understand rephrasing, so — unlike keyword SEO — you don't need ten versions of the same question. One prompt per distinct question. And no overlapping topics that split the same signal in two.

Each prompt counts toward your plan, so spend them on distinct questions (trials are capped at 200 prompts).
Set · 6

Clear ownership

The best test of a good topic: "Would a different team own the fix for this?" If yes, the grouping is right. Ownership is settled here, at the topic level — a prompt gets its owner from the topic it sits in.

A topic no team can act on is just a report line, not an opportunity.
Set · 7

Kept fresh

AI search moves fast. Drop prompts that never return a source, retire stale topics, and add new question patterns on a schedule rather than treating the set as done.

Practice: review the set every quarter. Cover the ground first, polish later.
Lens B

Prompt-level attributes — one prompt at a time

These are the things you can judge by looking at a single prompt. Five describe what a good prompt plainly is; the rest are quieter — how it behaves when run, where it's filed, and what happens if it scores badly — but they matter just as much. The most important of the five, being specific, comes down to one rule: give it a goal, a limit, and some context.

The three parts of a reliable prompt
01 Goal

What you want the model to do — help decide, not just explain.

Recommend the best option …
02 Limit

Who it's for or the situation it's in — narrows the field.

Recommend the best option for a small B2B marketing team on a tight budget …
03 Context

What it's being compared against — removes what the model would otherwise guess.

Recommend the best option for a small B2B marketing team on a tight budget — I need email automation with CRM built in, and I'm choosing between two tools
Put together — a prompt that runs the same way each time
"Recommend the best marketing automation tool for a small B2B team on a tight budget. I need email automation with CRM built in, and I'm choosing between two options — which should I pick?"
Prompt · 1

One task only

One question per prompt. Stack several in and the model quietly answers some and skips others, so the reading blurs. Split multi-part questions into separate prompts.

Prompt · 2

Real and natural

It should read like something a person would actually type or say — not a keyword string. "Best running shoes" is a search term; a real prompt sounds like a person with a need.

Prompt · 3

Your customers' words

Use the words your customers use and the phrasing and language of their market — not the product names only used inside your own company. The prompt should mirror how the audience really asks.

Prompt · 4

Aimed at a decision

Favour prompts tied to a real choice — "best…", "where to buy…", "…near me" — over idle "what is" questions. Each one should point to a page you'd want the answer to send people to.

Prompt · 5

Specific

The goal + limit + context above. The less the model has to guess, the tighter and steadier the answer. This is the part that does most of the work.

Prompt · 6

Self-contained

The prompt stands on its own. Because each one is run fresh and in isolation, over and over, it can't lean on an earlier message, a saved setting, or anything left unsaid — everything the model needs is in the prompt itself.

This, with Specific, is what keeps the answer steady run to run. No "as I said above", no loose ends.
Prompt · 7

Answerable

The prompt has to be able to return a source at all. Prompts that almost never surface any citation across many runs are dead ends — low value, and safe to drop. Being answerable is what makes a prompt worth acting on.

Watch how often it returns any source over repeated runs.
Prompt · 8

Correct and unique

No false assumption baked into the wording, and not a near-copy of a prompt you already have. Each prompt earns its place by testing something the others don't.

Prompt · 9

Filed under the right topic

A well-written prompt still hurts you if it's in the wrong bucket — a pricing question filed under how-to pulls both topics' scores off. Each prompt has to sit in the topic that matches what it's actually asking.

Placement is also what routes a poor result to the right team — the owner comes from the topic (Set 6).
Prompt · 10

Actionable

When a prompt scores poorly, the team should already know the move. Each one should be tied to something you can change — a page or piece of content to improve, an edge fix to deploy. A prompt whose bad result leaves everyone shrugging isn't worth tracking.

The who comes from the topic (Set 6); the what to do is wired in here.
Same question, two ways of writing it

Keyword style — noisy

"best PDF software"
No goal, no limit, no context. The model can take it a dozen ways, and the list of brands reshuffles each run.
"car dealerships"
A search target, not a question a person asks. There's no decision for the answer to help with.

Test-case style — steady

"I want a beginner-friendly PDF tool to turn files into Word and make quick edits — I'm comparing two options, which is simpler?"
Goal (choose), limit (beginner, simple), context (convert + edit, two options). Stands on its own and runs the same way each time.
"Which car dealership near central Singapore is best for servicing a family SUV under warranty?"
Real intent, local wording, a decision the answer can actually help with.

What quietly breaks a set

Common failure modes

Each one traces back to a missing attribute above.

Whiteboard prompts

Prompts invented in a room instead of drawn from real query data. They measure what you imagine, not what people ask. → Gate 0

One-note set

A set that's all Defend (or a branded/unbranded ratio copied from a template) with no goal behind the mix. → Set 3, tiers

Keyword thinking

Dropping search keywords straight in as prompts. They give the model no real question to answer, so results scatter. → Prompt 2, 5

The vague prompt

"Best CRM" with no frame around it. The model fills the gaps differently each run, so movement is noise, not signal. → Prompt 5

Copy-paste variants

Ten rewordings of one question, keyword-SEO style. Burns budget without adding coverage — the model already handles rephrasing. → Set 5

Dead-end prompts

Prompts that almost never return a source. They can't move and shouldn't drive decisions — drop them. → Prompt 7

Thin topics

A topic with three prompts. One stray answer swings the whole trend; not enough to trust. → Set 4

Topics no one owns

A topic no team is responsible for. Even a real gap goes unfixed, because there's nowhere for the work to land. → Set 6

Misfiled prompts

A prompt in the wrong topic — a pricing question under how-to. It drags two topic scores off at once. → Prompt 9

No follow-through

A prompt that can score poorly but isn't tied to anything you can change, so a bad result just sits there. → Prompt 10
