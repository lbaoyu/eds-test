LLMO Agentic Traffic Filters

Reference for the Agent Type and User Agent filters in the Agentic Traffic dashboard. This page consolidates the documented filter options, explains what each option means, and clarifies the practical difference between the two filters.

Agent Type = category User Agent = specific normalized agent label Current documented set

Overview

In the LLMO Agentic Traffic dashboard, the two filters serve different purposes:

Agent Type

A behavioral category that groups traffic into high-level classes such as Chatbots, Training bots, Research, or Web search crawlers.

User Agent

A specific normalized user-agent name that LLMO stores after matching raw CDN log user-agent strings, such as ChatGPT-User, GPTBot, or Gemini-Deep-Research.

1) Agent Type filter — all documented options and meanings

These are the normalized categories LLMO uses to group recognized agentic user agents.

Agent Type option Meaning Typical examples tracked by LLMO
All Agents No filter; includes every recognized agent category. All recognized user agents
Training bots Automated agents that gather content for long-term large language model training. GPTBot, ClaudeBot
Web search crawlers Bots that index site content to power search engines and LLM search functionality. OAI-SearchBot, PerplexityBot, Claude-SearchBot, Google
Chatbots Bots that retrieve content on demand to generate responses, answer questions, or provide citations. ChatGPT-User, Perplexity-User, Claude-User, Amzn-User, Google-ai-mode, GoogleAgent-URLContext, MistralAI-User
Research Specialized agents designed for deeper, task-specific investigation and synthesis. Gemini-Deep-Research, Google-NotebookLM
Media fetchers Agents that retrieve media assets such as favicons, images, or similar resources. ChatGPT Atlas, ChatGPT Clients, Perplexity Clients
Action agents Agents that perform explicit, goal-driven actions inside browsers, tools, or external systems. GoogleAgent-Chrome, GoogleAgent-Mariner
Shopping agents Agents focused on shopping-oriented tasks and product-related actions. GoogleAgent-Shopping
Ads Bot Agents that validate the safety of web pages submitted as ads. OAI-AdsBot

Important note: one internal Slack summary referenced a narrower subset of dropdown options. The broader and more current taxonomy comes from the newer LLM Optimizer: Metrics page plus the Agentic user agents (LLMO) reference.

3) Practical difference between the filters

Filter What it does
Agent Type Groups traffic into broad behavioral categories like Chatbots, Training bots, Research, or Web search crawlers.
User Agent Lets you isolate the specific normalized agent identity, such as ChatGPT-User, GPTBot, ClaudeBot, or Gemini-Deep-Research.

Short version: Agent Type = category, User Agent = specific bot label.

4) Caveats worth knowing

  • The recognized user-agent list is expanded over time as new LLM agents appear, so this is the currently documented set, not a guaranteed final set forever.
  • Some UI behavior and platform mapping are handled on the UI side. For example, an internal engineering thread notes that GoogleAgent-Chrome is currently classified under Gemini at the platform level.

5) Executive summary

If you need a fast interpretation of the dashboard:

  • Agent Type options are the high-level buckets: All Agents, Training bots, Web search crawlers, Chatbots, Research, Media fetchers, Action agents, Shopping agents, Ads Bot.
  • User Agent options are the specific normalized names like: GPTBot, OAI-SearchBot, ChatGPT-User, PerplexityBot, Perplexity-User, Gemini-Deep-Research, ClaudeBot, Claude-User, and others.
