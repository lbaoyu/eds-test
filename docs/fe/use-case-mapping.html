What it is

An advisory engagement that turns capability into outcomes

Help the customer develop, prioritise, and align their AEM Sites business use cases and KPIs to specific product capabilities — creating a clear roadmap that drives measurable outcomes and maturity growth. Every use case is connected to product features, cross-solution synergies, and ROI metrics.

The job to be done

Customers rarely lack ambition — they lack the line that connects “what AEM Sites can do” to “what the business is trying to achieve” and “how we’ll know it worked.” Your role is to draw that line and make it measurable.

The engagement is collaborative: customers do not need fully formed use cases beforehand. You expand early ideas into actionable value opportunities.

Why KPI-driven, and why AEM Sites

AEM Sites value is easy to assert and hard to prove. Anchoring the engagement to a measurable framework — Core Web Vitals, authoring throughput, availability, release velocity, ops effort — converts a feature conversation into a value conversation the customer’s leadership recognises.

This guide uses a five-category, Before-vs-After KPI model as the measurement spine for every mapped use case.

When to use it

Listen for these signals

This is the right accelerator when the customer has the platform but not the plan. Typical triggers:

You hear“We deployed the tool, but what’s next?”
You hear“We don’t know what we don’t know about the tool’s capabilities.”
You hear“Why did we purchase this?”
You hear“When do we use feature X vs. feature Y?”
You hear“We don’t have defined use cases.”
You seeLow adoption scores or unmet contractual usage.
You seeMaturity has stagnated and cross-solution opportunities are underleveraged.
How to deliver it

A six-phase model across 40 hours

A suggested structure for the time-boxed engagement. Adjust the split to the customer’s maturity, but keep the through-line: discover → develop use cases → develop KPIs → map to capability → value map & next-best-action.

0

Scope & prepare

~4h · pre-work

Confirm the AEM Sites scope with the CSM, qualify the signals, and gather inputs before the first workshop: adoption/usage data, contractual entitlements, current analytics and Core Web Vitals, known pain points, and read access to AEM Sites Optimizer and Cloud Manager. Set the agenda and stakeholder list.

Output: Engagement charter, input pack, confirmed stakeholders and agenda.
1

Discover & assess maturity

~6h

Working session to capture current business objectives, how AEM Sites is actually being used, and where the platform is on the maturity curve. Surface pain in the customer’s own words and tie each to a business cost (slow campaigns, poor CWV, outages, high ops effort).

Output: Maturity baseline and a prioritised pain inventory mapped to business impact.
2

Use Case Development Workshop

~8h

Diverge then converge: reframe pain points and ambitions into candidate AEM Sites use cases, then prioritise them on impact × effort. Expand early ideas rather than filtering them — the point is to widen the customer’s view of what the platform can do.

Output: Prioritised use case backlog (must/should/could).
3

KPI Development Workshop

~8h

For each priority use case, define measurable outcomes using the five-category KPI framework. Agree a baseline (from real Sites Optimizer / RUM / Cloud Manager / Analytics data where available, illustrative if not), set a target, and name the data source, owner, and review cadence.

Output: Prioritised KPI measurement framework with baselines, targets and owners.
4

Map capabilities & best practices

~8h

Tie each use case to specific AEM Sites capabilities and, where relevant, cross-solution synergies. Resolve the “feature X vs. feature Y” questions explicitly, and document feature best practices tied to the defined use cases.

Output: Use Case → Capability → KPI mapping with best-practice notes.
5

Value map & next-best-action

~6h

Assemble the Use Case & Value Map and the KPI framework into a single narrative, sequence a next-best-action plan aligned to roadmap priorities and maturity growth, and run the executive readout.

Output: Final Use Case & Value Map, KPI framework, and prioritised next-best-action roadmap.
The measurement spine

Five outcome-oriented KPI categories

Every mapped use case should land in at least one of these. For each KPI, define a baseline on the current state, a target on AEM as a Cloud Service, and a measured “after” — giving the customer a clean Before-vs-After story rather than a list of features.

Site Performance & Experience

Example KPIs
  • Core Web Vitals — % “good” views (LCP, CLS, INP)
  • % of top pages “good” on all three CWVs
  • Indexed coverage, organic sessions, search CTR, avg. position
Direction

Faster, smoother experiences and better discoverability.

Facilitation

Running the two workshops

These are the engine of the engagement. Both are collaborative working sessions — you facilitate and expand, you don’t prescribe or build.

Use Case Development Workshop

Reframe each pain point and ambition as an outcome, then a candidate use case. Use a jobs-to-be-done lens to keep it business-led, and prioritise on impact × effort. Capture more than you’ll action — breadth is the value the customer can’t generate alone.

FE tip: when a customer says “we don’t have use cases”, that’s the workshop, not a blocker. Start from what hurts and what they wish were faster.

KPI Development Workshop

For every priority use case, attach at least one KPI from the five-category framework. Set baseline, target, data source, owner and cadence. Use the customer’s real telemetry where you can; where you can’t, mark baselines as illustrative and note how they’ll be captured.

FE tip: a use case without a KPI is an opinion. Don’t let one leave the room unmeasured.

Prioritised KPI Framework

Unlock compound value

Cross-solution synergies

AEM Sites delivers more when it doesn’t stand alone. Identify these synergies in the engagement to expand the value map — but remember the scope is advisory and AEM Sites-led: you surface the opportunity, you don’t implement it.

Measure & understand

Analytics / CJA for engagement, conversion and journey analysis on top of CWV. AEP & Real-Time CDP for the audiences that make site experiences relevant.

Personalise & orchestrate

Target for experimentation and personalisation of AEM Sites pages. Adobe Journey Optimizer for orchestrating the journeys those pages sit within. AEM Assets / Dynamic Media for the content that feeds them.

Stay in lane

Guardrails

!What this engagement is — and isn’t

  • Advisory only. No hands-on configuration, code implementation, troubleshooting, or ongoing optimisation.
  • Time-boxed to 40 hours of Adobe effort. Scope the workshops to fit, and park execution-level detail for the appropriate follow-on motion.
  • Not for customers who already have fully developed use cases and are seeking execution or optimisation — route those to the right accelerator or services engagement.
  • AEM Sites is the focus here. Cross-solution synergies are identified to enrich the value map, not implemented within this engagement.

Who delivers it

Global Field Engineering, often in partnership with the CSM during the use case development phase. Customers do not need fully developed use cases before the engagement — the process is collaborative and helps expand early ideas into actionable value opportunities.
