AEM as a Cloud Service – Google Tag Gateway / First-Party GTM Implementation

Technical Implementation Guide

This document outlines a recommended implementation pattern for deploying Google Tag Gateway / first-party Google Tag Manager delivery on AEM as a Cloud Service.

The goal is to load Google Tag Manager from a first-party path on the site domain rather than directly from a third-party Google endpoint. A typical example path is /metrics.

1. Solution Overview

The implementation has two parts:

  1. CDN / infrastructure routing to proxy a first-party website path to Google Tag Gateway.
  2. AEM page integration to load the GTM container from that first-party path.

On AEM as a Cloud Service, request routing behavior such as origin selectors, request transformations, and related CDN rules is configured at the Adobe-managed CDN layer and deployed through a configuration pipeline (Configuring Traffic at the CDN).

Google documents the first-party delivery model as Google tag gateway (Google tag gateway for advertisers, Set up Google tag gateway for advertisers).

2. Architecture Summary

Layer Responsibility Notes
CDN / Infrastructure Route first-party path such as /metrics/* to Google Tag Gateway origin Includes required header handling and forwarding behavior.
AEM Render GTM script and noscript iframe using configuration-driven values No hardcoded values in multiple templates.
Analytics / GTM Team Manage tags, triggers, and rules in Google Tag Manager Most future tag changes can be handled in GTM.

3. Operational Outcome

After the base implementation is live, most ongoing changes can be managed directly in Google Tag Manager without updating AEM code.

Examples that typically do not require AEM code changes:

  • adding new marketing or advertising tags
  • changing trigger logic
  • updating firing rules based on URL patterns or existing DOM selectors
  • using values already available in the page or data layer

Examples that do require AEM development:

  • introducing new data points not currently rendered by the page
  • adding new interaction events not currently exposed
  • adding new business metadata for analytics
  • adding new consent or customer-state signals not already available
Change Type AEM Change Needed?
Add a new tag using existing page conditions No, in most cases
Adjust GTM trigger logic using existing DOM or data layer values No, in most cases
Change the GTM container ID No code change; configuration update only
Change the first-party path No code change; AEM configuration and CDN rule update only
Track a new custom component interaction not currently instrumented Yes
Expose new business metadata to GTM Yes

4. Recommended AEM Pattern

The recommended implementation pattern is:

  • OSGi configuration stores the GTM container ID and first-party path.
  • A Java service reads the OSGi configuration.
  • A Sling Model exposes the values to HTL.
  • head.html renders the GTM bootstrap script.
  • body-open.html renders the noscript iframe immediately after the opening body tag.
  • Rendering is enabled only on publish-facing output.

Adobe's recommended server-side templating technology for AEM is HTL (Getting Started with HTL). Client-side assets and related packaging patterns are documented in Using Client-Side Libraries on AEM as a Cloud Service.

5. Example Assumptions

  • GTM container ID: GTM-ABC1234
  • First-party path: /metrics
  • Expected first-party endpoints:
    • /metrics/gtm.js
    • /metrics/ns.html
    • /metrics/healthy

7. OSGi Service Implementation

TrackingServiceImpl.java

package com.myproj.core.services.impl;

import org.apache.commons.lang3.StringUtils;
import org.osgi.service.component.annotations.Activate;
import org.osgi.service.component.annotations.Component;
import org.osgi.service.metatype.annotations.AttributeDefinition;
import org.osgi.service.metatype.annotations.Designate;
import org.osgi.service.metatype.annotations.ObjectClassDefinition;

import com.myproj.core.services.TrackingService;

@Component(service = TrackingService.class)
@Designate(ocd = TrackingServiceImpl.Configuration.class)
public class TrackingServiceImpl implements TrackingService {

    @ObjectClassDefinition(name = "My Project GTM Tracking Configuration")
    public @interface Configuration {

        @AttributeDefinition(name = "Enable GTM")
        boolean enabled() default false;

        @AttributeDefinition(name = "GTM Container ID")
        String containerId() default "";

        @AttributeDefinition(name = "First-party GTM path")
        String firstPartyPath() default "/metrics";
    }

    private boolean enabled;
    private String containerId;
    private String firstPartyPath;

    @Activate
    protected void activate(Configuration config) {
        this.enabled = config.enabled();
        this.containerId = StringUtils.trimToEmpty(config.containerId());
        this.firstPartyPath = normalizePath(config.firstPartyPath());
    }

    @Override
    public boolean isEnabled() {
        return enabled;
    }

    @Override
    public String getContainerId() {
        return containerId;
    }

    @Override
    public String getFirstPartyPath() {
        return firstPartyPath;
    }

    private String normalizePath(String value) {
        String path = StringUtils.trimToEmpty(value);

        if (StringUtils.isBlank(path)) {
            return "";
        }

        if (!path.startsWith("/")) {
            path = "/" + path;
        }

        if (path.endsWith("/") && path.length() > 1) {
            path = path.substring(0, path.length() - 1);
        }

        return path;
    }
}

8. Sling Model for HTL

TrackingConfig.java

package com.myproj.core.models;

import org.apache.commons.lang3.StringUtils;
import org.apache.sling.api.SlingHttpServletRequest;
import org.apache.sling.models.annotations.DefaultInjectionStrategy;
import org.apache.sling.models.annotations.Model;
import org.apache.sling.models.annotations.injectorspecific.OSGiService;

import com.myproj.core.services.TrackingService;

@Model(
    adaptables = SlingHttpServletRequest.class,
    defaultInjectionStrategy = DefaultInjectionStrategy.OPTIONAL
)
public class TrackingConfig {

    @OSGiService
    private TrackingService trackingService;

    public boolean isGtmEnabled() {
        return trackingService != null
            && trackingService.isEnabled()
            && StringUtils.isNotBlank(trackingService.getContainerId())
            && StringUtils.isNotBlank(trackingService.getFirstPartyPath());
    }

    public String getGtmContainerId() {
        return trackingService != null ? trackingService.getContainerId() : "";
    }

    public String getGtmFirstPartyPath() {
        return trackingService != null ? trackingService.getFirstPartyPath() : "";
    }

    public String getGtmIframeSrc() {
        if (!isGtmEnabled()) {
            return "";
        }
        return getGtmFirstPartyPath() + "/ns.html?id=" + getGtmContainerId();
    }
}

10. HTL Integration

head.html

<sly data-sly-use.tracking="com.myproj.core.models.TrackingConfig" />

<sly data-sly-test="${wcmmode.disabled && tracking.gtmEnabled}">
    <script>
        window.dataLayer = window.dataLayer || [];
        (function(w,d,s,l,i,p){
            w[l]=w[l]||[];
            w[l].push({
                'gtm.start': new Date().getTime(),
                event: 'gtm.js'
            });
            var f=d.getElementsByTagName(s)[0],
                j=d.createElement(s),
                dl=l !== 'dataLayer' ? '&l=' + l : '';
            j.async = true;
            j.src = p + '/gtm.js?id=' + encodeURIComponent(i) + dl;
            f.parentNode.insertBefore(j, f);
        })(
            window,
            document,
            'script',
            'dataLayer',
            '${tracking.gtmContainerId @ context="scriptString"}',
            '${tracking.gtmFirstPartyPath @ context="scriptString"}'
        );
    </script>
</sly>

body-open.html

<sly data-sly-use.tracking="com.myproj.core.models.TrackingConfig" />

<sly data-sly-test="${wcmmode.disabled && tracking.gtmEnabled}">
    <noscript>
        <iframe
            src="${tracking.gtmIframeSrc @ context='uri'}"
            height="0"
            width="0"
            style="display:none;visibility:hidden">
        </iframe>
    </noscript>
</sly>

page.html Example

<!DOCTYPE html>
<html lang="en">
<head>
    <sly data-sly-include="head.html"></sly>
</head>
<body>
    <sly data-sly-include="body-open.html"></sly>

    <main>
        <sly data-sly-resource="${'root' @ resourceType='wcm/foundation/components/responsivegrid'}"></sly>
    </main>
</body>
</html>

11. CDN / Infrastructure Requirements

The AEM code above assumes the first-party route is already working at the CDN layer. The infrastructure configuration should:

  • reserve a first-party path such as /metrics
  • route /metrics/* to the Google Tag Gateway origin
  • set the required request headers documented by Google
  • forward cookies and query strings
  • validate the health endpoint

Relevant documentation:

  • Google tag gateway for advertisers
  • Set up Google tag gateway for advertisers
  • Configuring Traffic at the CDN
  • CDN in AEM as a Cloud Service

12. Deployment Sequence

Step 1 – Configure CDN Routing

  • Create the first-party route.
  • Point it to the Google Tag Gateway origin.
  • Apply the required header and forwarding logic.
  • Validate /metrics/healthy.

Step 2 – Deploy AEM Code

  • Add the Java service interface.
  • Add the OSGi service implementation.
  • Add the Sling Model.
  • Add the OSGi configuration files per environment.
  • Update head.html and body-open.html.
  • Deploy through the standard Cloud Manager application pipeline.

13. Validation Checklist

HTML Validation

  • Confirm page source loads GTM from /metrics/gtm.js?id=GTM-ABC1234.
  • Confirm the noscript iframe points to /metrics/ns.html?id=GTM-ABC1234.

Network Validation

  • Confirm requests are made to the site's own domain.
  • Confirm requests use the first-party path.
  • Confirm no direct GTM bootstrap request to googletagmanager.com remains.

Infrastructure Validation

  • Confirm https://www.example.com/metrics/healthy responds correctly.
  • Confirm required gateway headers are present at the proxy layer.
  • Confirm cookies and query strings are forwarded as expected.

14. Risks and Common Mistakes

Risk Impact Mitigation
AEM path does not match CDN path GTM fails to load Keep OSGi configuration aligned with CDN routing.
Only the script is updated and not the noscript iframe Implementation is incomplete Update both head.html and body-open.html.
Tracking is enabled on author Unwanted author-side requests and noise Disable in author configuration and render only on publish-facing output.
Future GTM use cases need data not currently exposed by the page Analytics team becomes dependent on new development Introduce or expand a structured site data layer.
Values are hardcoded in templates Higher maintenance effort Use centralized OSGi configuration.

15. Recommendation

A production-ready implementation should follow this pattern:

  1. Use CDN configuration for all Google Tag Gateway routing behavior.
  2. Use configuration-driven AEM HTL for rendering the GTM snippet.
  3. Keep the GTM container ID and first-party path externalized in OSGi configuration.
  4. Optionally add or strengthen a site data layer to support future analytics growth without repeated site changes.

This approach keeps day-to-day tag management in GTM while preserving a clean AEM implementation and a clear boundary between infrastructure, site code, and analytics operations.
