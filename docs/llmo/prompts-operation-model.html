Core Product Principles

1. Prompts are test cases, not keywords

Prompts should behave as repeatable measurement instruments rather than loose SEO-style phrases.

2. Discovery and tracking are different jobs

The tool should allow broad discovery, but reserve tracking for fewer, more intentional prompts.

3. The library is an asset, not a dump

A prompt library should preserve knowledge, provenance, ownership, and reuse value.

4. Stability matters for measurement

Tracking sets should not churn constantly or the before/after optimization signal gets weaker.

Most important operating rule: do not let the tracking set become a brainstorming scratchpad.

Recommended End-to-End Workflow

Step 1 — Build the Universe

Start broad using Semrush signals, GSC, AI visibility trends, site content, competitor language, and team input.

Step 2 — Curate into the Library

Retain the prompts worth keeping. Deduplicate, tag, classify, assign topic and intent, and preserve source metadata.

Step 3 — Create a Research Set

Pull a focused subset into a working space where analysts can refine, cluster, score, compare, and decide what moves forward.

Step 4 — Promote a Tracking Set

Move only the strongest prompts into active monitoring for visibility trends, opportunity generation, and impact reporting.

Step 5 — Review on Cadence

Revisit prompts regularly but deliberately. Add or retire prompts based on evidence, not churn.

In One Sentence

Discover widely, store carefully, research selectively, and track only what is strategic enough to measure over time.

What to Encourage vs What to Prevent

Encourage

  • Ground prompts in real demand and real customer language
  • Group prompts by intent and topic
  • Keep the library broader than the tracking set
  • Use research sets for refinement before promotion
  • Maintain stable baselines for measurement windows
  • Version prompt changes and record rationale

Prevent

  • Uploading every brainstormed prompt directly to tracking
  • Mixing unrelated intents into one topic
  • Frequent changes before or after optimization windows
  • Keyword-stuffed or robotic prompts
  • Over-indexing on branded prompts only
  • Treating the live prompt bank as a temporary scratchpad
Main platform risk: paraphrase inflation — library growth without distinct new signal.
