Agentic Traffic Guide

Understanding the Agent Type and User Agent Filters

This guide explains the two most important filters often used when analyzing AI-driven traffic: Agent Type and User Agent. It is written for a customer audience and focuses on what each option means in plain language.

Agent Type = broad category User Agent = specific bot or agent name Customer-facing explanation

Overview

These two filters answer different questions:

Agent Type

Helps you understand what kind of AI behavior you are looking at, such as training, search crawling, chat-driven retrieval, research, or task execution.

User Agent

Helps you identify the specific AI bot or agent making the request, such as GPTBot, ChatGPT-User, ClaudeBot, or PerplexityBot.

1) Agent Type — what each option means

Agent Type groups AI traffic into broad categories based on the role the agent is performing.

Agent Type option What it means Typical examples
All Agents Shows all recognized AI-related agents together, without narrowing to one category. All known AI agents and bots
Training Bots Agents that collect publicly available content to help improve future AI models. GPTBot, ClaudeBot
Web Search Crawlers Agents that discover and index content so it can appear in AI-assisted search experiences. OAI-SearchBot, PerplexityBot, Claude-SearchBot
Chatbots Agents that fetch content in response to a live user question so the AI system can answer or cite a source. ChatGPT-User, Perplexity-User, Claude-User, MistralAI-User
Research Agents Agents designed to perform deeper, multi-step investigation and synthesis across sources. Gemini-Deep-Research, Google-NotebookLM
Media Fetchers Agents that request images, icons, or other media assets rather than general page content. Client or media-fetching variants used by AI platforms
Action Agents Agents that can take goal-driven actions in tools or browsers on a user's behalf. GoogleAgent-Chrome, GoogleAgent-Mariner
Shopping Agents Agents that support shopping-related tasks, comparisons, or product-focused actions. GoogleAgent-Shopping

Simple way to think about it: Agent Type tells you the job the AI system is doing, not just the name of the bot.

2) User Agent — what each option means

User Agent shows the specific bot or agent identity associated with the request.

User Agent option Provider Typical role Plain-language meaning
GPTBot OpenAI Training Bot Used to collect public web content that may help improve future AI models.
OAI-SearchBot OpenAI Web Search Crawler Used for search-related discovery and retrieval.
ChatGPT-User OpenAI Chatbot Used when ChatGPT fetches content on behalf of a user question.
PerplexityBot Perplexity Web Search Crawler Used to index and discover content for Perplexity's search and answer experience.
Perplexity-User Perplexity Chatbot Used when Perplexity fetches live content to answer a user's question.
ClaudeBot Anthropic Training Bot Used to collect public content for model improvement.
Claude-SearchBot Anthropic Web Search Crawler Used to index public content for search-style answering.
Claude-User Anthropic Chatbot Used when Claude visits a page because a user asked it to retrieve content.
Gemini-Deep-Research Google Research Agent Used for deeper, multi-step research tasks that gather and synthesize information.
MistralAI-User Mistral AI Chatbot Used when Mistral retrieves live web content during a user conversation.
GoogleAgent-Chrome Google Action Agent Used when a Google AI agent performs browser-based or tool-based actions.
GoogleAgent-Mariner Google Action Agent Used for task-oriented browsing and execution workflows.

Simple way to think about it: User Agent tells you which exact AI bot made the request.

3) How these two filters differ

Filter Best used for
Agent Type Understanding the kind of activity taking place, such as training, search discovery, live answer generation, or task execution.
User Agent Understanding which specific bot was responsible for the activity.

In short: Agent Type explains the category of behavior, while User Agent identifies the individual bot or agent behind that behavior.

4) Helpful notes

  • Not every provider uses only one bot. A single provider may use different agents for training, search, live retrieval, media access, or task execution.
  • New AI agents appear over time, so available options may expand as platforms introduce new capabilities.
  • If you are reviewing traffic trends, filtering by Agent Type usually gives the clearest strategic view first. Filtering by User Agent is best when you want detail about a specific provider or bot.

5) Quick summary

Use Agent Type when you want to answer:

  • Are these requests coming from training bots, chatbots, or search crawlers?
  • How much of my AI traffic is research-oriented versus user-driven retrieval?

Use User Agent when you want to answer:

  • Was this traffic from ChatGPT, Claude, Perplexity, Gemini, or another provider?
  • Which exact bot was active on the site?
