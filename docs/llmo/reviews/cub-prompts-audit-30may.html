04 The issues to fix

Seven items, roughly in order of priority. The first two are blocking; the rest sharpen measurement quality.

1 · Over the trial cap

The trial allows 200 active prompts; the file has 268, so it's 68 over and can't go in as-is. This is the first thing to correct — see the cleanup plan in Section 07.

2 · Topic and prompt fields are swapped

In 172 of 268 rows, the actual user question is stored in the Topic field while the Prompt field holds a generic placeholder ("個人金融"). The library isn't reliably separating category → topic → prompt.

Left unfixed, topic reporting is noisy, rollups mislead, and duplicate prompt families are hard to spot. Remap so each row is: Category (business area) → Topic (stable use-case family) → Prompt (the specific natural-language question).

3 · No brand aliases configured

No own-brand aliases are set up in LLMO — neither the brand (Cathay United Bank, 國泰世華) nor the products (CUBE, 國泰 Cube, Cube 卡, cathay-cube). Two consequences: the system can't reliably tell which prompts are branded; and when an AI answer refers to Cathay as "CUBE" or another variant, those Brand Mentions and Share of Voice go uncounted — so the branded side under-counts.

This is a non-trivial problem, but a quick fix: register the brand and main product aliases (in both Chinese and English, with common spellings) in LLMO. Once configured, the branded/unbranded split and brand metrics become accurate — and this is a prerequisite if the goal is to focus on CUBE's performance (see Section 05).

4 · Too many keyword fragments

Around 80 entries are SEO-style keyword fragments rather than questions a person would actually ask an assistant — for example "CUBE新戶優惠", "優惠存款利率", "線上貸款怎麼辦", "頂級卡推薦哪張", "美金定存優惠比較". They're too broad and variable for stable measurement. A strong prompt usually carries a goal, a context, a constraint, and often a comparison frame (see Section 10).

5 · Inconsistent topic taxonomy

Several labels point to the same family — e.g. "信用卡 機場接送" / "信用卡機場接送" / "機場接送 信用卡", and "國泰cube" / "國泰cube卡". This false fragmentation spreads volume across duplicate topics and makes pruning and comparison harder. Normalise to one canonical name per family.

6 · Market metadata mismatch

All 268 rows are marked Market = US, but the library is overwhelmingly Traditional Chinese and Taiwan-focused (cards, deposits, loans, local behaviour). This is almost certainly wrong metadata; if the monitoring market is Taiwan, set it accordingly.

7 · Unfinished & off-domain rows

The 14 processing rows shouldn't be part of a final library. They cluster into four topics — business cards (商務卡), hotel card-payment (酒店可以刷卡嗎), USD–RMB currency conversion (2000美元是多少人民币), and generic account-opening (开户). Several are off-domain or China / US-oriented and unsuitable for a Taiwan card trial. Remove them first.

10 Rewrite guide

In LLMO a prompt is a test case — a repeatable, customer-style question used to track how the assistant answers a real need. If the prompt is weak, the signal is weak. Here's what "strong" means, and how to upgrade the weak ones.

Five qualities of a strong prompt

QualityWhat it meansBanking example
NaturalSounds like a real customer question.國泰世華 CUBE 卡海外刷卡回饋多少？
ClearIntent is obvious on the first read.申辦 CUBE 卡需要準備哪些文件？
SpecificPoints to a real need or use case.常出國的人適合哪一張國泰信用卡？
FocusedAsks one main thing, not several.CUBE 卡的機場接送怎麼預約？
RepeatableMeans the same thing if asked again later.國泰世華數位帳戶的活存利率是多少？

A simple formula — User + Need + Context

A guide, not a strict rule. When a prompt feels too thin or too broad, adding the user and the context is usually the quickest way to strengthen it.

U

User · who is asking?

首辦卡新戶 · 常用行動支付的人 · 常出國的人 · 小資族 · 小型企業負責人

N

Need · what are they doing?

比較 · 申辦 · 查回饋 · 海外消費 · 開戶 · 配置外幣

C

Context · what makes it clearer?

在台灣 · 不想付國外手續費 · 每月固定加油 · 短期美元配置 · 與台新／永豐比較

Good ingredients
A market or location context · a life stage or use case · a spending pattern · a clear product need · a comparison set · a decision constraint.
Five ways prompts go weak
Too short (a label, not a question) · too broad (could mean anything) · too many asks (several needs at once) · too vague (no real intent) · too unnatural (internal wording, not customer language).

Remember — the goal isn't longer prompts, it's stronger ones. A strong prompt is natural, clear, specific, focused and repeatable. Often that means adding a user and a bit of context, but a short prompt that already passes those tests is fine as it is. Don't pad prompts just to make them longer, and don't stack several questions into one.
